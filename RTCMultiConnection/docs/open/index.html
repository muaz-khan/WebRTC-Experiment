RTCMultiConnection API Reference / "open" method

  1. open will be called only for initiator!
  2. open method allows you open a new room in the current signaling channel:
  3. open method accepts object parameter as well; also now it returns "sessionDescription" object: 
    // you can pass object instead on string!
if(you_are_initiator) {
    var sessionDescription = connection.open({
        dontTransmit: true,    // optional --- disable default room-forwarding behaviour!
        sessionid: 'something', // optional
        transmitRoomOnce: true // optional
    });
}

// you can share "sessionDescription" object using socket.io or any gateway
// and target users can directly join using "join" method:
if(you_are_participant) {
    connection.join( sessionDescription );
}

  4. if(you_are_initiator) {
    connection.open('session-id'); // i.e. room-id
}

// or
if(you_are_initiator) {
    connection.sessionid = 'session-id';
    connection.open();
}
  5. If you skip "session-id"; RTCMultiConnection will use "full-URL" as session-id: 
    var sessionid = location.href.replace(/\/|:|#|%|\.|\[|\]/g, '');
connection.open(sessionid);
  6. "sessionDescription" object that is returned by "open" method allows you "join" on demand, anytime, using any style!
  7. You can disable default room-forwarding mechanism by passing "dontTransmit:true" parameter.
