RTCMultiConnection API Reference / "captureUserMedia"

  1. You can use captureUserMedia to manually capture media streams.
  2. It accepts two arguments; first one is mandatory and last one is optional:
    1. success-callback: which is called on successfully getting user media resources.
    2. session: which allows you force custom media type to be captured.
  3. var session = {
    audio: true
};

connection.captureUserMedia(function (mediaStream) { }, session);
  4. captureUserMedia is useful when initiator is sending "audio+video" stream; however, participant want to share "audio-only" stream. 
  5. initiator.session = {
     audio: true,
     video: true
};

initiator.open();

// for participant

participant.onNewSession = function(session) {
     var session = {
         audio: true
     };

     connection.captureUserMedia(function (mediaStream) {
          // Remember, call "join" method in success-callback!
          connection.join(session);
     }, session);
};
  6. captureUserMedia is also useful when initiator is sending "audio+screen" stream in oneway direction and participant wants to add "audio" stream in his answer.
  7. It will act as "OneWay Screen & Two-Way Audio".
